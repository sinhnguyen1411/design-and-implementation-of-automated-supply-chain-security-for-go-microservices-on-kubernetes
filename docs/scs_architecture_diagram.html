Thesis Architecture View

Supply Chain Security Architecture for Go Microservices on Kubernetes

This diagram presents the end-to-end control chain defined by the thesis baseline: dependency integrity, vulnerability gating, signed artifact publication, provenance attestation, and Kubernetes admission enforcement with auditable evidence collection.

Scope: Go microservice + Kubernetes
Trust model: verify before release, enforce before run
Evidence: CI logs, attestation, admission events

Pre-release control

The CI path validates Go dependency integrity, runs govulncheck, generates an SBOM with Syft, scans risk with Grype, and blocks publication when policy is violated.

Artifact trust

Only artifacts that pass the pipeline are signed with Cosign and accompanied by SLSA-style provenance, making image origin and build lineage verifiable before deployment.

Runtime enforcement

Kubernetes admission applies trust policy at deploy time, rejecting unsigned or non-compliant workloads and preserving decision evidence for the thesis validation report.

Presentation note: this view is intentionally thesis-facing. It emphasizes control objectives and enforcement points, while implementation details remain documented in README.md and docs/devsecops_ci_admission.md.
